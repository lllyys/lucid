# Feature #23 — Clear button on the polish input pane

Single-WI (final) acceptance. A Clear button on the polish Original pane, mirroring the translate source Clear;
design `dev-docs/designs/lucid-polish-clear/` (closed needs-design #199). Shipped v0.22.0 (PR #209, merge `fde7ebe`).

## Environment
- Built the production app (`pnpm build` → `dist/`, v0.22.0); served via a token-free single-origin
  `@lucid/server` (port 8804, `/tmp` DB). Headless Chrome via raw CDP; fresh profile. Typed into the polish
  Original textarea via the React-controlled native value-setter + `input` event. Detection scoped precisely to
  the OriginalCard (its "meaning reference" subtitle) to isolate it from the translate source Clear.

## Acceptance criteria

| # | Criterion | Observed | Result |
|---|---|---|---|
| 1 | Clear is **hidden** when the Original is empty | No Clear button in the OriginalCard when empty | ✅ pass |
| 2 | Clear **appears** when the Original has text | After typing, the OriginalCard shows Clear (leading ⌕ LOOKUP + 中文 picker — screenshot) | ✅ pass |
| 3 | Clicking Clear **wipes** the Original input | Original textarea value `"我们需要…初稿。" → ""` | ✅ pass |
| 4 | After Clear, the button **hides again** + the Original textarea is **refocused** | Clear gone; `document.activeElement === the Original textarea` (the design's refocus) | ✅ pass |
| 5 | Clear is placed **leading** the header right-group (before LookupToggle + LanguagePicker) | Header renders "ORIGINAL · meaning reference" left; "Clear · ⌕ LOOKUP · 中文 ▾" right (screenshot) | ✅ pass |
| 6 | Clear does **NOT** fire an LLM re-polish (the M1 guard), even under auto-run | Non-arming `clearOriginal` (no `armPolish`); covered by the PolishPanel test (auto-run armed + pending → advance 1500ms → `polish.status` idle) + Gate-4 audit | ✅ pass (tests + audit) |

## Observations
- The OriginalCard header (screenshot `/tmp/lucid-23-shown.png`) renders faithfully to the design — Clear leads
  the right control group (before ⌕ LOOKUP + the language picker); the empty DRAFT-TO-POLISH card below
  correctly shows no Clear, confirming the `value.trim()` gating is scoped per-input.
- A first CDP pass reported PARTIAL due to a too-broad button-matching heuristic (an 8-ancestor walk that
  matched the translate source Clear via a shared container); re-running with detection scoped to the
  OriginalCard's "meaning reference" subtitle gave a clean PASS. The feature was correct throughout — the
  heuristic was the fault.
- The M1 guard (Clear must not fire an LLM call) is a non-visual behavior; it's covered by the PolishPanel unit
  test + the Gate-4 audit (the non-arming `clearOriginal` traced against the runId/debounce machinery), not by
  the browser pass.
