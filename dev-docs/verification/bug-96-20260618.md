# Bug #96 verification — polish result shows only the polished text

**Close-gate path:** verification-exception. The failure cannot be deterministically browser-reproduced
in this environment — it is triggered by a *non-compliant model* emitting wrapper prose ("Here is the
improved sentence:" + surrounding quotes + a "Changes made:" list), which requires that specific model
plus the user's API key (held in memory only, rule 65 §5 — unavailable to the verifier). The default
Anthropic provider obeys the hardened prompt and emits no prose to clean, so a live browser polish
cannot exhibit the failure on demand. A deterministic regression at the **real component + store + diff
+ cleaner boundary** drives the exact code path the production failure hits.
