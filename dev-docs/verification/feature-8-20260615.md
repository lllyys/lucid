# Feature #8 — Persist polish keywords: acceptance evidence

The polish keyword set now persists globally (one set across sessions) via the same crash-proof
`safeJSONStorage` the glossary and sessions already use. WI-1 is the only WI (first + final).
Verified by composition — the storage round-trip is proven against an injected in-memory backend in
`safeJSONStorage.test.ts`; this WI proves the store wires that mechanism correctly (`lucid.keywords`,
version 1, migrate + partialize) and that the keyword behavior is unchanged. No provider/network
involvement.

## Method

- `pnpm check:all` = `lint` → `typecheck` → `test:coverage` (100% on logic globs) → `build`. Green.
- Store/helper unit tests; persistence mechanism proven via the injected-backend round-trip in
  `safeJSONStorage.test.ts` (the vitest env has no real localStorage — confirmed by the
  `--localstorage-file` warning). No provider/live-API involvement (rule 66 §4).
- Gate-2 plan audit: independent subagent, 1 round (Critical/High from a false localStorage-exists
  premise, fixed in plan v2). Gate-4 impl audit: independent subagent, CLEAN/ship-as-is —
  `.claude/codex-audits/feat-feature-8-wi-1-persist-keywords-audit.md`.

## Deferred

- **Live-browser / Playwright reload E2E** — Playwright is not a project dependency (named, confirmed
  absent), consistent with features #2/#3/#4 which verify persistence the same compositional way. The
  single-reload manual check (add a keyword in the Polish panel → refresh → it's still there) is
  offered to the user against `pnpm dev`; the mechanism it would exercise is already covered above.
- **Per-session keyword sets** — out of scope; global set shipped. A future feature row if requested.
- **Cross-tab live sync** — single-tab scope (reload to pick up another tab's set), matching the
  sessions/glossary persistence scope from feature #3.
