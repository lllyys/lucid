# Feature #10 — Multiple custom (OpenAI-compatible) providers

Final-WI acceptance pass. The feature turns the single-custom slot into N user-defined OpenAI-compatible
providers (label / base URL / model / optional key + per-custom connection test), each addable, editable,
removable, activatable, and switchable from the toolbar. Verified two ways: a real-browser CDP pass of the
live add flow + the designed surfaces rendering, and the mocked-provider RTL/store suite (the project's
determinism path — rule 65 §8) for the full behavior. The 4 WIs merged as one coherent batch (the data-model
change isn't independently shippable — Gate-4 WI-1 High).

## Acceptance criteria
