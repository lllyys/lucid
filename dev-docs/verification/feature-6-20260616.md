# Feature #6 — Verify API key / test connection: acceptance evidence

Delivered as part of the multi-provider umbrella (feature #5): the headless probe (WI-5) + the Settings
test-connection panel (WI-6b). A "Test connection" makes ONE minimal authenticated call through the
provider interface and reports valid / invalid / unreachable inline, with latency. Verified via unit +
behavioral tests with `fetch` mocked; never runs in `pnpm check:all` against a live API (rule 65 §8).

## Acceptance criteria → evidence

| Criterion | Result | Evidence |
|---|---|---|
| A minimal authenticated probe via the provider interface (not a vendor SDK) | pass | `testConnection.test.ts` — `probeProvider` uses the RAW single-attempt `provider.stream()` ({translate,'ping','en'}), times the first byte, then releases the request. |
| Reports valid / invalid / unreachable distinctly | pass | `testConnection.test.ts` maps 401/403→invalidKey, 429→rateLimited, 5xx→providerDown, timeout, in-stream refusal/incomplete, abort→aborted; `useTestConnection.test.ts` records `{status:'ok',latencyMs}` vs `{status:'fail',msgKey:'error.<kind>'}`. |

| No key logging; cost/timeout guarded; never in `check:all` | pass | Probe surfaces only latency + an ErrorKind-derived msgKey (never the key/raw error — rule 65 §5); bounded by `DEFAULT_PROBE_TIMEOUT_MS`; mocked at the fetch boundary in all tests. |

## Method

- Probe + hook unit tests (fetch mocked / createProvider mocked at the boundary); behavioral Settings
  tests via ARIA queries with a stubbed global fetch for the "Connected" path. No live API.
- Gate-4: WI-5 adversarial 3-lens workflow (CLEAN after a test-strength round); WI-6b adversarial 3-lens
  workflow (design Medium — stat-grid order — fixed; CLEAN round-2). Logs in `.claude/codex-audits/`.

## Deferred

- **Live-API probe verification** — never run in `check:all` (cost/nondeterminism/key safety); a manual
  `pnpm dev` test against a real key is offered to the user. The probe's behavior is covered by the
  mocked-boundary tests.
- **Cancel affordance for an in-flight test** — accepted: the probe is bounded by a timeout and writes
  to the argument-captured vendor (no stale write); a per-vendor AbortController is a future nicety.
