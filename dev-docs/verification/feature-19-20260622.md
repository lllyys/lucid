# Feature #19 — Unify workspace-data sync onto the single-origin server (final WI-3: simplified Settings·Sync UI)

Final-WI acceptance. #19 unifies sync onto the single-origin server and drops the separate token/URL config
for the common case. WI-1/WI-2 (server token-free `/sync` mode + client `connectSingleOrigin`) shipped in
v0.14.1; **WI-3 (this WI)** is the simplified Settings·Sync UI: an on/off toggle that connects token-free to
the app origin, with the manual URL/token form demoted behind an Advanced disclosure. Verified by a real-
browser CDP run against a token-free `@lucid/server` + the unit/component suite (1483 tests, 100% gated
coverage) + the design-gated Gate-2 (the design landed) and diff-scoped Gate-4 (ship-as-is) audits.

## Observations
- The toggle connects token-free to `window.location.origin` (the served origin), exactly the #19 model — no
  URL/token entry needed for the single-origin (Tailscale) case the feature targets.
- Backward compat is the load-bearing risk and it holds: remote (non-empty-token) users keep the server row +
  Edit; only the token-free path renders the read-only origin row. Guarded by a unit test.
- The `error.syncSession` "Sign in" banner was deferred rather than invented (no store state surfaces it;
  #16 owns the global banner surface) — rule-51-faithful.
- Residual: the deferred session-expired banner is a follow-up if/when the store surfaces an auth/session
  state; the >500 kB chunk warning is pre-existing.
