# Feature #26 — Tighter editor resting height

Final-WI acceptance for the 1-WI feature (`min-h-[88px]`→`min-h-[56px]` textarea, `min-h-[130px]`→
`min-h-[98px]` polish card, via the shared `src/lib/editor/editorSizing.ts` constants). Design bundle
`dev-docs/designs/lucid-editor-resting-height/` (issue #219).

**Verified commits:** feature code `8b15b97` (WI-1) + fix `1cbd390` (the first-keystroke-jump fix).
Branch `feat/26-editor-resting-height`; verify-time HEAD `1cbd390`. Not yet merged; #26 version bump
pending (PR tail commit).

**Result: PASS.** This is a re-run after the prior round's single FAIL. The prior run passed 6/7 and
**FAILED check 3** — the Translate Source jumped 34px down (empty 91px → one-line 57px) on the first
keystroke, because the 71-char source placeholder wrapped to 2 lines and, with `field-sizing: content`,
sized the empty textarea to 2 lines. Fix `1cbd390` shortened `translate.sourcePlaceholder` to the
design's one-line copy **"Type or paste text…"**. This run confirms **the jump is gone** (empty 57px ==
one-line 57px, delta 0) and re-confirms the other six criteria. A phone-width (390px) spot-check across
all three editors was added: **no jump at phone width either.**

## Deferred
- **Built-CSS grep after `pnpm build` (plan DoD / Gate-2 Top-Risk 1).** Verified in dev, not the
  production bundle. The Tailwind-content-scan risk (that moving the `min-h-[56px]`/`min-h-[98px]` literals
  into a `.ts` constant would stop them being emitted) is nonetheless **resolved**: live
  `getComputedStyle().minHeight` returned `56px`/`98px`, which only happens if Tailwind emitted the rule
  AND it applied to the element — stronger than a static CSS grep. The literal `pnpm build` + `grep` step
  runs at the merge gate via `pnpm check:all`.
- **Phone tier full grow-to-cap.** The phone spot-check exercised resting height + no-jump for all three
  editors and confirmed the 50vh cap value (`maxHeight 422px`); it did not paste a long document to hit the
  phone cap live (desktop cap already exercised at 88vh). The `min-h-[56px]` constant is tier-agnostic
  (design: "same rule, no phone fork"), so phone resting height is 56px as measured.
