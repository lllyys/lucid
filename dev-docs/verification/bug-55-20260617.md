# Bug #55 verification — globally-unique entity ids

**Close-gate path:** verification-exception. The failure cannot be browser-reproduced in this
environment — Playwright's browser binary is not installed, and the cross-device half of the bug
needs two concurrent clients. A deterministic regression at the **real store boundary** (the actual
`useSessionStore` / `useGlossaryStore` actions + the real production id generator) drives the same
code path the production collision would hit.

## Acceptance criteria → evidence

| Criterion (expected behavior) | Result | Evidence |
| --- | --- | --- |
| A session/task id minted after a reload does not collide with a pre-existing id | pass | `sessionStore.test.ts` › "mints collision-free session ids across reloads" — `__useRandomSessionIds()` drives the prod generator, re-initialized between two `newSession()` calls (simulating a reload); asserts the second id ≠ the first and matches the full `s_<v4-uuid>` shape. Under the old counter this re-issued `s1`. |
| Task ids are likewise globally unique | pass | `sessionStore.test.ts` › "mints collision-free task ids in production too" — two `addTask` ids differ and match `t_<v4-uuid>`. |
| Glossary term ids are likewise globally unique across reloads | pass | `glossaryStore.test.ts` › "mints collision-free term ids across reloads" — `g_<v4-uuid>`, distinct across a re-init. |
| Minting an id never throws in a non-secure context (self-hosted lucid over `http://` on a LAN, where `crypto.randomUUID` is undefined) | pass | `uuid.test.ts` › "falls back to getRandomValues in an insecure context" — stubs `crypto` to `{ getRandomValues }` (no `randomUUID`); `randomUuid()` returns a valid RFC-4122 v4. |
| Existing behavior unchanged — deterministic ids still available for tests | pass | All prior store/component tests pass unchanged; `__resetSessionIds`/`__resetGlossaryIds` install a deterministic counter (`s1`,`t2`,`g1`). |
| Cross-device uniqueness (a #9-sync prerequisite) | pass (by construction) | uuid v4 ids are globally unique, so two devices cannot mint the same id — the merge-by-id corruption path is eliminated. Not separately browser-tested (needs two clients). |
