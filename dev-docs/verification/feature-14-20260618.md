# Feature #14 verification — auto-save every completed translate/polish run to history

**Close-gate path:** verification-exception. The trigger — a *completed* translate/polish run — requires
a working provider/key, which is held in memory only (rule 65 §5) and is unavailable to a headless
browser, and no local Ollama is running; per rule 65 §8 `pnpm check:all` must never hit a live API. The
deterministic high-fidelity verification is the **panel integration tests**, which render the REAL
`TranslatePanel`/`PolishPanel` + the REAL `operationStore` + `sessionStore` + the REAL
`useAutoRecordTask`/`recordRunIfNew`, driving a run through a mocked provider transport to its `done`
transition — the exact production code path. Not stubs of the subsystems under test.
