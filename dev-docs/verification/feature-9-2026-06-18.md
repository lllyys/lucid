## Method

- **Real client↔server round-trip (data path):** built `@lucid/server` (`cd server && pnpm build`), ran
  `SYNC_TOKEN=test123 DB_PATH=/tmp/lucid-sync-acc.db PORT=8787 node dist/index.js`, then drove the exact
  REST contract the client's `createRestSyncBackend` uses via a node + global-`fetch` script (16 assertions
  across 8 groups: auth, push/apply/monotonic-rev, pull round-trip, conflict, restful update, incremental
  pull, purge, unreachable) — **ALL PASS**. Persistence-across-restart verified by pushing, restarting the
  server on the same durable SQLite file, and re-pulling.
- **Browser render smoke (UI mount):** served the app with `pnpm dev` and rendered it in the cached
  headless Chromium (`chrome-headless-shell-1217`, `--headless --dump-dom`). The rendered DOM contained the
  mounted sync pill ("Local-only", "not syncing", accessible name "Sync status: Local-only") alongside the
  Lucid brand + Translate/Polish panels — confirming the WI-9d shell wiring works in a real browser with no
  shell regression.
- **Component + headless logic (hermetic):** `pnpm check:all` — every sync component (pill, connect form,
  connected panel, status card, disconnect dialog, conflict card, settings panel/dialog, error banner) and
  the `Workspace` mount are RTL-tested for render + interaction; the merge/pull/push/cycle/orchestrator/
  controller engine is unit-tested with a mock backend (rule 65 §8). Provider/network never hit live in the
  suite.

## Deferred

- **Two-real-browser-tabs live sync E2E** — the project has no Playwright test harness (no dependency, no
  `playwright.config`, no `e2e/` dir); wiring one is out of scope for this verification. The equivalent was
  verified through the real-server round-trip (push from one client → pull from a second client) plus the
  headless-Chromium render smoke of the mounted UI. Per rule 47 Gate 5, the specific tool (a Playwright
  harness) is named and confirmed absent; the data path and UI mount are verified by other means above.
- **Live LLM provider calls** — never run in `pnpm check:all` (rule 65 §8); out of scope for the sync
  feature (sync transports workspace data only, not provider traffic).
