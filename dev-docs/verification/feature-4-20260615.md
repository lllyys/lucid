## Method

- `pnpm check:all` = `lint` → `typecheck` → `test:coverage` (100% on logic globs) → `build`. Green.
- Behavioral component tests (`@testing-library/react` + `userEvent`, ARIA-role queries) and the
  provider-stack integration test run against a **mocked** provider/`fetch` (rule 65 §8). No live API.
- Dark-theme AA ratios computed against the committed palette (WCAG 2.1 relative luminance).
- Gate-4 independent Codex audit: 3 rounds (`019ec974`→`019ec98c`); bidi converged on authoritative
  UCD `DerivedBidiClass` classification; log at `.claude/codex-audits/feat-feature-4-gap-surfaces-audit.md`.

## Deferred

- **Live-API end-to-end** — never in `pnpm check:all` (rule 65 §8); requires the user's own key
  entered via the now-shipped Settings dialog (#13). The full key→provider→fetch path is tested
  against a mocked fetch.
- **Browser / Playwright E2E** — Playwright is not a project dependency (only a stray global
  chromium from another project); wiring it is a separate decision. Behavioral ARIA tests cover the
  surfaces; consistent with feature #2's mock-verified approach.
- **bidi-L non-letter edge** — accepted Low (see the Gate-4 audit log): a rare bidi-class-L
  non-letter before RTL text would be skipped rather than treated as first-strong-L; negligible for
  a Chinese↔English tool.
