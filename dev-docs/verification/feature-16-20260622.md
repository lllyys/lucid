# Feature #16 — Responsive / mobile layout (resolves #17)

Final-WI acceptance. The workspace reflows across three tiers (desktop ≥960 unchanged · tablet 600–959
drawer+stacked · phone <600 drawer + single-pane Translate/Polish switcher), fixing the phone overflow from
the triage screenshot. Verified by a real-browser CDP run at all three tiers + the unit/integration suite
(1467 tests, 100% gated coverage) + the 4-lens Gate-2 (2 rounds) and diff-scoped Gate-4 (ship-as-is) audits.

- The phone overflow (the original triage screenshot) is gone — `scrollWidth == clientWidth` at 390px.

  toggling visibility — the round-trip test proves source text + a partially-rejected diff survive.
- The 4-lens Gate-2 Workflow caught 1 Critical + 7 High + 7 Med before any code; the diff-scoped Gate-4
  (after the 4-lens Gate-4 Workflow stalled on the large worktree read) confirmed ship-as-is.

  hamburger/gear are the design's drawn 34px (WCAG-min compliant); real-device iOS safe-area not fully
  modellable in headless CDP.
