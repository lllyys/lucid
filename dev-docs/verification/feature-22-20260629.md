## Observations
- The "★ Starred" tab + list + type badges + lang pairs render faithfully to the design §C (screenshot
  `/tmp/lucid-22-starred.png`). Search filters by both halves; Unstar removes.
- The star *buttons* themselves only show on a `done` lookup/translation; with no provider configured the
  define/translate requests don't reach `done`, so the live "click the star on a real result" path isn't
  CDP-exercisable here (the #20-documented headless provider-form friction). The toggle/build/reflect logic is
  covered by the StarButton/WordLookupPopover/TranslateResult/PolishResult component tests + the Gate-4 audit,
  and the store→review round-trip is shown live via the seeded Starred tab.
- **Deferred (tracked follow-up):** the design's word-detail "Open in workspace ›" + the detail IPA "Speak word"
  button (need workspace-load / speech wiring) — filed as a follow-up so the rule-51 surface is eventually
  complete; the core review contract (translation/IPA/meaning/From/back/Unstar) is met.
