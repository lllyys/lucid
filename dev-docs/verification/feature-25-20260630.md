# Feature #25 — Read a recorded session task

Final-WI acceptance for the 4-WI feature (Task metadata capture → sync round-trip → `TaskReadView` →
interactive row). Design `dev-docs/designs/lucid-session-task-read/` (closed needs-design #212). Shipped v0.23.0
(PR #216, merge `676129a`). Resolves the user report "sessions are restored but cant be read".

## Environment
- Built the production app (`pnpm build` → `dist/`, v0.23.0); served via a token-free single-origin
  `@lucid/server` (port 8805, `/tmp` DB). Headless Chrome via raw CDP; fresh profile. Seeded
  `localStorage['lucid.sessions']` (persist v2) with one session holding a **translate** task (CJK source +
  result + `sourceLang:zh`/`targetLang:en`/`durationMs:1500`) and a **polish** task (original + polished +
  `keywords:['inference','attention mechanism','token']`).

## Observations
- The translate read view (screenshot `/tmp/lucid-25-translate.png`) renders faithfully to the design's Part B —
  back link, the ⇄ header with the full **direction + latency + age** metadata, Source above / Result below, and
  the sticky Copy + Open-in-workspace row. The metadata proves the full WI-1 (capture) → WI-2 (sync round-trip)
  → WI-3 (render) chain works end-to-end (the audit's decisive H1 concern — that sync would clobber the new
  fields — is resolved).
- The polish read view (screenshot `/tmp/lucid-25-polish.png`) renders Part C — Original + Polished + the
  Keywords-kept chips.
- The row `↗` correctly **loads without opening the read view** (the H3 sibling-buttons + stopPropagation
  decision verified live): the body opens the read view, the `↗` loads into the editor — two distinct actions,
  no bleed.
- A first CDP pass reported PARTIAL on CRIT3 only because the assertion expected the translate task's CJK source
  but the first `↗` in DOM loaded the polish task's English source (both valid "a task's source loaded"); a
  targeted re-run against the translate task's `↗` gave a clean PASS.
- The age "…ago" reads large because the seed used `createdAt: 5` (≈ epoch) — a test-data artifact, not a bug;
  `formatAge` renders the relative age correctly from `createdAt`.
- Open-in-workspace lands in **translate** for both kinds (the documented #24-consistent default; a
  polish-Original-targeted load is a noted follow-up).
