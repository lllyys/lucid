## Observations
- **Load order (expected):** on a server that also exposes `/config`, the **#15 config-sync gate** ("No synced
  config yet — Set a passphrase / Keep working local-only") renders first; the workspace (and therefore the
  #21 consent) mounts only after that gate is resolved. So a fresh token-free single-origin user sees the
  config gate, then — on dismissing it — the workspace-sync consent. This is the two-sync split (#15 keys vs
  #9/#19 workspace data) surfacing in sequence, not a #21 defect; #21's consent behaves correctly once the
  workspace mounts. (The eventual two-sync unification is the separately-tracked, user-declined option from the
  2026-06-28 triage.)
- Accept hands off to the existing #9 status pill (no "Got it" card) exactly as the design annotates; Esc =
  decline; no quiet dismiss leaves the question open.
