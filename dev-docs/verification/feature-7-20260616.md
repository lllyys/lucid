# Feature #7 — Custom / OpenAI-compatible provider: acceptance evidence

The custom provider is fully delivered: the engine (`openaiCompatibleStream`, shipped v0.4.1 — #7
WI-1/2/3) plus the Settings configuration UI (base URL + model + an OPTIONAL key), which was the
design-gated WI-4 and landed in feature #5's WI-6a after the #29 design loop. Users can point Lucid at
any OpenAI-compatible endpoint — keyless self-hosted (LM Studio, vLLM, local) OR a keyed proxy
(OpenRouter). Verified via unit + behavioral tests with `fetch` mocked.

## Method

- Provider + store unit tests with `fetch` mocked; behavioral Settings tests via ARIA queries. No live API.
- Gate-4: #7 engine WI-1/2/3 audited CLEAN (`.claude/codex-audits/feat-feature-7-custom-provider-audit.md`);
  the Settings fields (WI-6a) audited via an adversarial 4-lens workflow (the keyless-vs-keyed decision +
  the `usePanelRun` run-path fix), CLEAN round-2.

## Deferred

- **Live custom-endpoint verification** — never in `check:all`; a manual `pnpm dev` pass against a real
  endpoint (e.g. a local LM Studio or OpenRouter) is offered to the user.
- **Non-localhost is fully supported for custom** (it IS the base-URL field); only Ollama's base URL is
  fixed to localhost (a separate future needs-design if a remote Ollama is required).
