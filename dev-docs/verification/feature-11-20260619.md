# Feature #11 — Auto-run translate & polish (debounced auto-trigger)

Final-WI acceptance pass. The feature adds an opt-in, debounced auto-run that reuses the manual run
pipeline (`usePanelRun` + `AbortController`), with an API-cost gate, CJK/IME safety, and readiness gating.
Verified two ways: a real-browser smoke (the designed UI renders + is readiness-gated) and the
mocked-provider RTL/store suite (the interactive behavior — rule 65 §8 / rule 66 §4 mandate a mocked
provider over live calls for determinism).

## Acceptance criteria

| # | Criterion (from the #11 row) | How verified | Result |
|---|------------------------------|--------------|--------|
| 1 | Debounced auto-run fires after the input settles, reusing the run pipeline | RTL: typing arms a 1.5s pending timer that then fires `run(panel, req, isAuto=true)` (`TranslatePanel.test`/`PolishPanel.test` "debounced typing fires an auto run") | pass |

| 4 | Hosted-provider cost gate on first enable; local (Ollama) skips it | RTL: hosted first-enable opens `AutoRunCostDialog` (accept → ack + enable); ollama enables directly | pass |
| 5 | Cost gate re-checked on a vendor switch (no surprise paid auto-runs) | RTL: switching to an unacked hosted vendor while enabled → `armed=false` (auto-fire suppressed) + cost dialog re-opens (Gate-4 M1 fix, 5 tests) | pass |
| 6 | CJK/IME: never fire mid-composition; re-arm on commit | RTL: `useAutoRunDebounce` holds during `compositionstart`, re-arms on `compositionend`; PolishPanel guards the draftTranslate mirror | pass |

## Observations

- The browser confirms the **rendered** reality the RTL suite can't: both panel headers mount the designed
  toggle, both are correctly disabled with no provider key (readiness gating, design Section D), and the
  pending-ring CSS keyframe shipped — with zero console errors.
- The streaming-dependent criteria (AUTO tag on a completed run, suppression, IME, ⌘↵) are verified against
  a **mocked** provider per the project's determinism rules rather than a live LLM — the appropriate fidelity
  for these paths, not a deferral.
- The Gate-4 M1 fix (cost re-gate on a vendor switch) is the load-bearing rule-65 safety property: a user who
  enabled auto-run on local Ollama and later switches to a hosted vendor cannot trigger surprise paid calls.
