# Feature #18 — Polish-goal selector

Final-WI acceptance. A single-select goal-chip row (Clarity / Grammar / Tone / Concise) in the polish
header lets the user choose how the draft is refined; the choice flows into the polish request (was
hardcoded `'clarity'`). Verified by a real-browser CDP render/interaction check plus the unit/integration
suite (1383 tests, 100% gated coverage) and the two-round Gate-2 plan audit + ship-as-is Gate-4 audit.

## Observations

- The chips render in the polish header (the closest analog to the design's shared control bar, which the
  lucid stacked-panel layout never implemented — documented in the plan).
- Single-select is an accepted v1 deviation from the multi-select design: the goal instructions conflict
  when combined (`grammar`: "do not otherwise rewrite" vs the rewriting goals), so combining them would
  produce an incoherent prompt. Multi-goal is a documented follow-up.
- The H1/H2 regression tests genuinely fail if the wiring regresses (the auto-run test asserts the armed
  request carries the *new* goal via a capturing provider, not stale state).
